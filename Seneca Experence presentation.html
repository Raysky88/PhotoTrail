The Birth of Processing (2001)

[translate:Ben Fry] and [translate:Casey Reas], researchers at MIT Media Lab, created Processing to make programming fun for artists and designers.

  • Built on Java for visual graphics and interaction
  • Designed with simple, intuitive syntax
  • Let creatives "sketch" with code

Processing.js (2008) - Bringing Code to the Web

[translate:John Resig], creator of jQuery, built a transpiler that converts Processing code to JavaScript

  • Runs in web browsers without plugins
  • Opens HTML5 Canvas for interactive graphics
  • Seneca students contributed significantly to this project

Seneca's Contribution

Seneca students played a major role in advancing Processing.js, especially the 3D components

  • Applied research partnership with Seneca College
  • 3D graphics and advanced features development
  • Bridging art and computer science education

The Interactive Photo Frame Project

A web-based interactive application combining photography, animation, and web technologies

  • Live demo: catherine-leung.github.io/photoframe/
  • Fully customizable with code changes
  • Real-world example of web programming

Technologies Used

Frontend

  • HTML5 Canvas
  • JavaScript (ES6+)
  • CSS3 Styling

Frameworks

  • Processing.js Library
  • Web APIs
  • SVG Graphics

How the Photo Frame Works

  • Loads and displays images dynamically
  • Responds to user mouse and keyboard input
  • Creates smooth animations and transitions
  • Completely interactive and responsive

Changing the Photo Frame - Simple Customization

Let's see how easy it is to modify the behavior with basic code changes: 

// Change frame background color
background(245, 232, 220);  // Beige color

// Adjust animation speed
let speed = 2;  // Increase from 1 for faster animations

Changing the Photo Frame - More Features

// Add frame border
stroke(50, 50, 50);
strokeWeight(8);

// Control rotation speed
rotationSpeed = 0.05;  // Slower rotation
